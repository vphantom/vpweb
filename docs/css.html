Vanilla+ CSS Framework

Introduction

There are already plenty of CSS frameworks, with varying levels of complexity. This one attempts to be a progressive enhancement on plain HTML5, with CSS based improvements with occasional JS components where necessary. It strays from the extreme simplicity of the flat layered "paper" trend in favor of more obvious classic visual cues enhanced for modern browsers.

All overrides, classes and custom styling on elements not prefixed with vp- only take effect inside .vp blocks to help reduce collisions with other frameworks. (See Coöperation below.)

Compiled CSS files are located in dist/. You may use vpweb.css to get the entire framework at once, or just some of the individual files described below. Be sure to import core.css first in order to avoid priority side-effects.

Coöperation

No special behavior on standard elements should leak outside of .vp blocks (which may be <html>, <body> or just small fragments of your pages). You may also disable any section or single standard elements with .xvp. We achieve this flexibility with: selector:is(.vp, .vp *):not(.xvp, .xvp *). Note that elements and classes with an explicit vp- prefix may retain their functionality despite .xvp because they are implicitly unique.

Build Process

Responsive

Certain classes restrict which devices can see an element, by viewport width or display type. Note that the names "mobile", "tablet" and "desktop" are used as nicknames for the three responsive size ranges, not as true device identification. CSS media queries use a "device pixel ratio" to define their own meaning for "px", so those are not actual pixels but industry standard values.

Mobile:
Less than 768px, .mobile-only, .tablet-down
Tablet:
From 768px to less than 992px, .tablet-only, .tablet-up, .tablet-down
Desktop:
From 992px up, .tablet-up, .desktop-only
Large Desktop:
From 1200px up, .largedesktop-only may occasionally be useful.
Pointer:
With no fine pointer, .mouse-only is hidden. When no pointer can hover, .hover-only is hidden.
Print:
On a screen, .print-only is hidden, while on any other device, .screen-only is hidden.

Grid grid.css

Grids are defined by adding attribute vp-grid="x" to an element, where x is one of: 8, 10, 12, 15 or 16. Cells in grids are immediate child elements, optionally with attribute vp-cols="x" where x is a number between 2 and 16 (1 being implicit). Rows wrap, so you only need to use multiple grid blocks for splitting incomplete lines or changing bases.

An <img> directly inside a [vp-grid] or wrapped in just an a automatically gets full width.

Caution: while spacing is sensitive to density, it also causes a small negative margin on the left and right ends of rows. Therefore, using a page-wide grid with non-tight density will cause horizontal scrolling. As long as a grid's container has at least as much spacing, it is not an issue. (i.e. A tight or spaced row fits inside a page-wide relaxed or spaced container.)

[vp-grid].doubling
Makes grid cells double in relative width when stepping down from "desktop" to "tablet" size, and double again down to "mobile" size. (i.e. A 1/12th cell would become 1/6th wide in "tablet" viewports and 1/3rd wide in "mobile" viewports.)

Forms forms.css

Note also that margins between some elements is affected by the form's density.

Fields label[vp-field] > input|select|textarea

In addition to text, typical input types compatible with this class are: email, number, password, tel, text, url and possibly search. Fields expand to fill their container's width, which is handy in grids. We implement infield top aligned labels for their clarity with the contents of the title attribute, for accessibility.

An optional .lead or .follow sibling of the input may precede it to provide icons. They are centered in a 1.5em wide area. Note that a .follow sibling must still come before the input and would not make sense for select or textarea.

Tooltips tooltips.css

For uniformity with touch devices, our tooltips are touch/click activated. This requires adding the ontouchstart attribute to body and adding tabindex="0" to non-clickable elements like span.

To evade their trigger or parent element's width, tooltips don't break on white space. Keep this in mind for placement to avoid overflowing the viewport. For element tooltips, you may insert manual breaks as desired.

Attribute [vp-tooltip]

To give any element (except inputs) a clickable tooltip, the plain text content of the tooltip need simply be supplied in a vp-tooltip="" attribute. Optional accompanying boolean attributes vp-tooltip-below and vp-tooltip-right affect alignment and position.

Click this sentence for example.

Element vp-tooltip

To deal with inputs and to use inline HTML in a tooltip, use vp-tooltip which must immediately follow a focusable element such as input, and both need to be inside a position:relative block. For our forms, this is easily achieved with a field (an input inside its label). Optional boolean attributes _below and _right affect alignment and position.
