Vanilla+ CSS Framework

Introduction

There are already plenty of CSS frameworks, with varying levels of complexity. This one attempts to be as small as possible, with pure CSS solutions where possible, without sacrificing esthetics which contribute to usability. It strays from the extreme simplicity of the flat layered "paper" trend in favor of more classic visual cues enhanced for modern browsers.

Almost all classes and custom elements begin with the prefix vp- to prevent collisions with other frameworks. For brevity, there is no prefix for attribute classes like is-something, has-something and size-something.

Compiled CSS files are located in dist/. You may use vpweb.css to get the entire framework at once, or just some of the individual files described below. Be sure to import core.css first in order to avoid priority side-effects.

Cooperation

Semantic CSS rules without a vp-prefix only apply when a parent has class vp. It is therefore easy to use Vanilla+ in only parts of a document. The only rules which apply outside are in reset.css: 


*, *:before, *:after {
	box-sizing: inherit;
	margin: 0;
	padding: 0;
}
html, body {
	height: 100%;
}
html {
	box-sizing: border-box;
	line-height: 1.33;
	-webkit-text-size-adjust: 100%;
}

If this interferes with your other CSS, you may use vpweb-noreset.css instead of vpweb.css but some borders, margins and paddings may be slightly off in some browsers.

Responsive

Certain classes restrict which devices can see an element, by viewport width or display type. Note that the names "mobile", "tablet" and "desktop" are used as nicknames for the three responsive size ranges, not as true device identification. CSS media queries also use a "device pixel ratio" to define their own meaning for "px", so those are not actual pixels.

Mobile:
Up to 767px, .vp-mobile-only, .vp-tablet-down
Tablet:
From 768px to 991px, .vp-tablet-only, .vp-tablet-up, .vp-tablet-down
Desktop:
From 992px up, .vp-tablet-up, .vp-desktop-only
Pointer:
With at least one fine pointer, .vp-touch-only is hidden. With no fine pointer, .vp-mouse-only is hidden
Print:
.vp-print-only, .vp-screen-only (non-screen is considered "print")

Block .vp-block

Makes any element (including buttons) a block and full width.

Generic Container .vp-container

The .vp-container class has side margins dictated by density (self or immediate parent) and limited width on large viewports.

Flex Layouts .vp-vfull, .vp-hfull

Applicable to any container block, including body. Sensitive to density setting.

.vp-vfull
2 or 3 children, stacked vertically. The second child expands as necessary to make sure that the optional third child is always no higher than the bottom of the viewport.
.vp-hfull
Same behavior, but horizontal from left to right. Set width using min-width to help with responsiveness. Responsive: On tablets down, the third child goes below the other two at 100% width and on mobile, all three children become vertically stacked.

Grid grid.css

Three bases (8, 10, 12) were chosen for their versatility: areas can be divided by 2, 3, 4, 5, 6, 8, 10 and 12.

Grids are defined by adding attribute vp-grid="x" to an element, where x is one of: 8, 10 or 12. Cells inside grids are elements with attribute vp-cell="x" where x is a number between 1 and 12. Rows wrap, so you only need to use multiple grid blocks for splitting incomplete lines or changing bases.

An <img> directly inside a [vp-grid] or wrapped in just an a automatically gets full width.

Caution: while spacing is sensitive to density, it also causes a small negative margin on the left and right ends of rows. Therefore, using a page-wide grid with non-seamless density will cause horizontal scrolling. As long as a grid's container has at least as much spacing, it is not an issue. (i.e. A seamless or spaced row fits inside a page-wide relaxed or spaced container.)

Responsive

.vp-doubling [vp-cell]
Makes grid cells double in relative width when stepping down from "desktop" to "tablet" size, and double again down to "mobile" size. (i.e. A 1/12th cell would become 1/6th wide in "tablet" viewports and 1/3rd wide in "mobile" viewports.)
.vp-stacking [vp-cell]
Makes grid cells go full-width for "tablet" and "mobile" sizes.

Forms forms.css

Note that buttons and input widgets with :focus get a z-index of 1000 or 1001 in addition to the halo. Note also that margins between some elements is affected by the form's density.

Fields label.vp-field .vp-label + input|select|textarea

In addition to text areas, typical input types compatible with this class are: email, number, password, tel, text, url and possibly search. Fields expand to fill their container's width, which is handy in grids. We implement infield top aligned labels for their clarity.

Tooltips tooltips.scss

For uniformity with touch devices, our tooltips are touch/click activated. This requires adding the ontouchstart attribute to body and adding tabindex="0" to non-clickable elements like span.

To evade their trigger or parent element's width, tooltips don't break on white space. Keep this in mind for placement to avoid overflowing the viewport. For element tooltips, you may insert manual breaks as desired.

Attribute [vp-tooltip]

To give any element (except inputs) a clickable tooltip, the plain text content of the tooltip need simply be supplied in a vp-tooltip="" attribute. Optional accompanying boolean attributes vp-tooltip-below and vp-tooltip-right affect alignment and position.

Click this sentence for example.

Element vp-tooltip

To deal with inputs and to use inline HTML in a tooltip, use vp-tooltip which must immediately follow a focusable element such as input, and both need to be inside a position:relative block. For our forms, this is easily achieved with a field (an input inside its label). Optional boolean attributes _below and _right affect alignment and position.

Transitions transitions.css

To guarantee that no browsers "flash" transitions on load, some rules depend on a parent having the "loaded" class. To use them, add the loaded class to body after the first paint is likely over. In some browsers, a simple "onload" is insufficient, thus:

<body onload="setTimeout(function(){document.body.classList.add('loaded')})">
